# Управление пользователями: навигация и связи модуля

## Назначение
Модуль `Управление пользователями` управляет web-пользователями, ролями, прямыми правами, сбросом паролей и ограничениями доступа по участкам.

## Где находится
- Sidebar: `Настройки -> Управление пользователями`.
- Прямой маршрут: `/#users`.
- Основные frontend-файлы: `SiteUsersScreen.tsx`, `components/site-users/*`, `siteUsersRepository.ts`.

## Экран и сценарии
- Список пользователей: поиск, статус, роли, последний вход.
- Правая панель пользователя: профиль, блокировка, сброс пароля.
- Панель прав: роли, direct permissions, effective permissions, scope-доступ по участкам.
- Для ЭМУ оператору можно назначить права работы и ограничить видимость конкретными участками.

## Backend и данные
- API namespace: `/api/v1/site-users`.
- Контроллер: `SiteUsersController`.
- Сервис: `ISiteUserAdminService`, `EfSiteUserAdminService`.
- Основные таблицы: `site_users`, `roles`, `permissions`, `site_user_roles`, `role_permissions`, `site_user_permissions`, `site_user_access_scopes`.

## Права доступа
- Управление пользователями закрыто правом `site_users.write`.
- Пользователь с `admin` ролью получает все права.
- Direct permissions накладываются поверх роли.
- Scope-доступ не выдает право сам по себе: он только ограничивает видимость данных внутри разрешенного модуля.

## Связи с другими модулями
- `Обход`, `Бухгалтерия`, `ЭМУ`, `PERCo-Web` используют permissions для доступа к экранам и действиям.
- `ЭМУ` дополнительно использует scope `emu_section`, чтобы ограничить оператора выбранными участками.

## Приемочные проверки
- Без `site_users.write` экран недоступен.
- Изменение direct permissions сохраняется и видно после перезагрузки.
- Scope `emu_section` ограничивает API ЭМУ, а не только интерфейс.
