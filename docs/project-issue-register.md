# Журнал проблемных мест Patrol360

Дата заведения: 2026-05-26  
Последнее обновление: 2026-05-28

Документ ведем как рабочий журнал аудита и ремонта проекта. Данные из рабочей PostgreSQL не исправляем "наугад": сначала фиксируем факт, симптом, место в коде/API/БД, причину, статус, следующий шаг и проверку.

## Формат записей

| Поле | Как заполнять |
|---|---|
| Модуль | Обход, мобильное приложение, ЭМУ, Бухгалтерия/Inventory, пользователи/RBAC, Dashboard, worker, Docker/infra |
| Приоритет | P0 - блокер, P1 - критично для рабочего контура, P2 - важно для качества/эксплуатации, P3 - низкий риск |
| Тип | ошибка, неправильная логика, неполная реализация, данные, инфраструктура, тесты, переделать |
| Статус | Исправлено, На доработку, Переделать, Требует решения, Подтверждено аудитом |
| Где искать | Экран, API, backend service, frontend file, mobile file, DB таблицы, тесты |
