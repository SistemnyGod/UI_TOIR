# Аудит визуальных дефектов интерфейса, 2026-06-28

## Проверенный фокус

Первый исправительный проход выполнен по экрану `#inventory-issue`, потому что по нему были самые конкретные дефекты: сломанная строка черновика, неудобные вкладки, кривой поиск, неочевидное добавление позиции и слишком крупный пустой блок сотрудника.

| Экран | Дефект | Исправление | Статус |
| --- | --- | --- | --- |
| `#inventory-issue` | Карточки номенклатуры требовали отдельную кнопку `Добавить` и выглядели как неактивные элементы | Карточка стала кликабельной целиком, с понятным hover/focus/selected состоянием | Готово |
| `#inventory-issue` | Строка черновика ломалась на малом экране: кнопки и поля съезжали | Закреплена сетка: предмет слева, количество/единица/цена/сумма по центру, действия справа | Готово |
