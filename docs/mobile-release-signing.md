# Подпись релизного APK

Пакет `ru.patrol360.mobile` закреплён за существующим сертификатом. Его публичный SHA-256 хранится в `mobiel proekt/release-signing.json`. Этот файл не содержит пароль или закрытый ключ и должен оставаться в Git: сборка использует его как защиту от случайной подписи другим сертификатом.

Закрытый ключ находится в `mobiel proekt/secrets/patrol360-release-v2.jks`. Каталог `secrets` исключён из Git. Для пароля существуют две локальные DPAPI-копии:

- `patrol360-release-v2.dpapi.json` — привязана к текущему Windows-пользователю;
- `patrol360-release.machine.dpapi.json` — привязана к этому компьютеру и работает при смене Windows-профиля.

## Проверка ключа без сборки

```powershell
cd 'mobiel proekt'
./scripts/build-release-apk.ps1 -ValidateOnly
```

Wrapper перебирает разрешённые metadata-файлы, расшифровывает доступный вариант и обязательно сверяет сертификат с `release-signing.json`. Повреждённая или устаревшая копия пропускается. Пароль не печатается и удаляется из переменных окружения после завершения.

## Сборка релизного APK

```powershell
cd 'mobiel proekt'
$env:PATROL360_ENVIRONMENT = 'local-enterprise'
./scripts/build-release-apk.ps1
```

Передача `-SecretPath` обычно больше не нужна. Она остаётся для диагностики и заставляет wrapper проверить только указанный metadata-файл.

## Повторное создание машинной DPAPI-копии

```powershell
cd 'mobiel proekt'
./scripts/backup-release-signing.ps1
```

Скрипт не перезаписывает существующую копию. Для осознанного обновления сначала сохраните старый файл вне рабочего каталога, затем удалите только `patrol360-release.machine.dpapi.json` и повторите команду.

## Защита от случайной ротации

`create-release-keystore.ps1` прекращает работу, если в проекте уже закреплён сертификат. Новый ключ нельзя использовать как обычную замену: Android не примет обновление ранее установленного приложения. Параметр `-AllowCertificateRotation` предназначен только для отдельно согласованной процедуры ротации сертификата.

## Аварийное восстановление

DPAPI-копия уровня компьютера защищает от смены профиля, но не от потери или переустановки самого компьютера. Для полного аварийного восстановления нужны две независимые внешние копии:

1. файл `patrol360-release-v2.jks` в защищённом корпоративном хранилище;
2. пароль JKS в менеджере секретов с разграничением доступа.

Не храните JKS и пароль вместе, не отправляйте их в чат и не добавляйте в Git. Публичный SHA-256 из `release-signing.json` следует использовать для проверки восстановленного ключа через `build-release-apk.ps1 -ValidateOnly`.
