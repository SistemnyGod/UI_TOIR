# Журнал изменений интерфейсов

Этот документ хранит краткие записи о точечных изменениях UI, адаптивном поведении, исправлениях отображения и небольших улучшениях логики.

## Правила ведения

- Одна запись — одно законченное изменение или небольшой связанный набор правок.
- Для сложной правки указывается точечная проверка, а не полный регрессионный прогон.
- Для простой визуальной правки достаточно сборки или ручной проверки затронутого экрана.
- Отдельно фиксируются ограничения и задачи, сознательно оставленные на следующий этап.

## 2026-07-23 — исправление SQLCipher-транзакций Android

- Исправлено открытие транзакционных соединений локальной базы: каждое новое соединение теперь получает тот же SQLCipher-ключ до выполнения SQL.
- Транзакции выполняются через защищённый helper с `BEGIN IMMEDIATE`, корректным `COMMIT`/`ROLLBACK` и закрытием соединения.
- Все репозитории мобильного приложения переведены с `db.withExclusiveTransactionAsync` на защищённый путь.
- Добавлена точечная регрессионная проверка, запрещающая возврат старого незащищённого вызова.

Файлы: `mobiel proekt/src/db/encryptedDatabase.ts`, `mobiel proekt/src/db/database.ts`, `mobiel proekt/src/db/repositories/*`, `mobiel proekt/tests/databaseEncryptionPolicy.test.ts`.

Проверка: `npm run verify` — 58 тестов пройдено, encoding/typecheck/lint без ошибок; standalone debug APK собран и содержит SQLCipher native libraries.

Ограничение: физическое Android-устройство не подключено, поэтому установка и запуск APK через adb не выполнены.
