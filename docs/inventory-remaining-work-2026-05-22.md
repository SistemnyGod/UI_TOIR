# Inventory: остаток переноса на 2026-05-22

## Что уже перенесено в Patrol360

- Базовый Inventory shell внутри текущего Atom/Patrol интерфейса без второго сайдбара.
- Основные разделы бухгалтерского модуля: обзор, сотрудники учета, номенклатура, выдача, операции, под запись, СИЗ, история, отчеты, настройки, пользователи, системный журнал.
- Backend-фундамент `/api/v1/inventory/*`: каталог, склады, остатки, движения, Custody, PPE, history, reports/export v1, legacy import v1.
- Экран `Под запись`: создание записи, журнал актов, строки ответственности, выбранный акт, PDF/DOCX действия и мини-предпросмотр.
- Экран `СИЗ`: журнал карточек, выбранная карточка, быстрые действия печати, мини-предпросмотр.
- Экран `Выдача`: глобальный подбор номенклатуры, черновик выдачи, выбор сотрудника и проведение выдачи несколькими строками через существующий backend.
- Настройки Inventory получили вкладки справочников, норм и наборов.

## Что осталось довести

- Проверить `INVENTORY_LEGACY_CONNECTION_STRING` на реальной PostgreSQL-схеме старого проекта и запустить dry-run import.
- Составить финальную mapping matrix: legacy table -> Patrol360 entity -> импортировано/пропущено/нужна ручная правка.
- Довести визуальные шаблоны печати:
  - личная карточка СИЗ;
  - лист подписи СИЗ;
  - акт под запись;
  - PDF/DOCX должны выглядеть как рабочие бланки, а не только v1-документы.
- Доделать детальные drawer:
  - история строки СИЗ;
  - история акта под запись;
  - состав акта с действиями close/open/archive;
  - карточка номенклатуры с остатками и быстрыми действиями.
- Доработать Inventory settings:
  - нормы СИЗ по должностям;
  - наборы и состав наборов;
  - причины возврата;
  - причины списания;
  - реквизиты организации для печати.
- Доработать роли и права Inventory поверх текущего RBAC Patrol360:
  - роли бухгалтерского модуля;
  - права на выдачу, возврат, списание, печать, импорт, настройки;
  - UI управления правами.
- Реализовать полноценный импорт сотрудников учета из Excel с preview результата.
- Довести отчеты:
  - остатки;
  - движения;
  - СИЗ;
  - под запись;
  - история;
  - сотрудники;
  - system log.
- Добавить интеграционные тесты на реальные сценарии:
  - Custody issue / return / writeoff / archive;
  - PPE create / issue / return / writeoff / cancel;
  - повторный legacy import без дублей;
  - PDF/DOCX/XLSX smoke.

## Первый шаг завтра

1. Задать `INVENTORY_LEGACY_CONNECTION_STRING`.
2. Запустить `POST /api/v1/inventory/legacy/import/dry-run`.
3. Сверить таблицы, counts и checksum остатков.
4. После этого закрывать пропуски importer-а и параллельно доводить печатные формы.
