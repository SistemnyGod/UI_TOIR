# Заметка по интерфейсу мобильного приложения Patrol360

Дата: 2026-05-28

Источник: макет с шестью экранами для обхода территории.

## Общие правила

- Авторизацию не менять: текущий экран входа оставляем как утвержденный.
- Основной стиль рабочих экранов: светлый фон `#F5F7FA`, белые карточки, мягкая тень, радиус 12 px, синий основной цвет `#1E5BFF`.
- На рабочих экранах нужен компактный верхний блок: логотип/название, статус сети `Онлайн/Оффлайн`, затем заголовок экрана.
- Нижняя навигация всегда остается выше системных кнопок Android.
- Пользователь не видит технический экран синхронизации. Состояние синхронизации показывается только в профильной карточке и рабочих ошибках.

## Экраны

1. `Обход`: главный экран с активным обходом, совместным аккаунтом, кнопками `Обновить заявки` и `Выбрать заявку`, карточкой текущей заявки.
2. `Выбор заявки`: верхние счетчики `Доступно`, `В работе`, `Назначено`, `Всего`; карточки заявок со статусами и переходом.
3. `Метки`: прогресс маршрута, счетчики, фильтры, список точек с порядком и статусом.
4. `Статус метки`: только выбор состояния `Исправно` / `Неисправно`; комментарий и фото на этом шаге не показывать.
5. `Комментарий и фото`: статус выбранной метки, комментарий, необязательное фото, кнопки камеры/галереи, действия `Отложить`, `Сохранить`, `Продолжить маршрут`.
6. `Профиль`: работы и замечания, синхронизация, устройство, настройки уведомлений/темы/языка, смена пользователя.

## Поведение

- Фото необязательное для `Исправно` и `Неисправно`.
- Для фото использовать системную камеру Android и галерею, не встроенный экран фотоаппарата.
- Метки можно проходить в любом порядке.
- Если нет интернета, данные сохраняются локально и отправляются после восстановления связи при открытом приложении.
